# LS-01: Listing Skor + Ücretsiz Revize

**Durum:** Açık
**Öncelik:** P1
**Tahmini süre:** 3-4 saat

---

## Amaç

Üretilen metin içeriğine kural bazlı skor vermek, eksik bilgileri önermek, düşük skorda ücretsiz revize hakkı tanımak. Sıfır ek AI maliyeti (skor), minimal maliyet (revize ~₺0,01).

## Skor hesaplama (rule-based, AI yok)

| Kural | Puan | Koşul |
|---|---|---|
| Başlık uzunluğu | 0-15 | Platform max limitine yakınlık |
| Özellik/madde sayısı | 0-15 | Platform beklentisi kadar varsa tam puan |
| Açıklama uzunluğu | 0-10 | Min 150 karakter = tam puan |
| Etiket sayısı | 0-10 | Platform limiti kadar varsa tam puan |
| Anahtar kelime kullanımı | 0-10 | Kullanıcının girdiği keyword'ler çıktıda var mı |
| Ürün detay zenginliği | 0-20 | Renk, malzeme, boyut, adet, ağırlık, garanti (her biri +4) |
| Kategori uyumu | 0-10 | Kategori seçilmiş + çıktıda yansımış mı |
| Fotoğraf kullanımı | 0-10 | Fotoğraf yüklenmişse +10 |
| **Toplam** | **0-100** | |

## Eksik bilgi önerileri

Skor hesaplanırken hangi kurallardan puan kaybedildiği kaydedilsin. Öneriler:
- Renk eksik → "Ürünün rengini ekle — alıcılar renk bilgisi arar"
- Malzeme eksik → "Malzeme bilgisi ekle — %100 pamuk, paslanmaz çelik vb."
- Boyut/adet eksik → "Boyut veya adet bilgisi ekle — 500ml, 3'lü set vb."
- Fotoğraf yok → "Fotoğraf ekle — metin kalitesi önemli ölçüde artar"
- Keyword eksik → "Anahtar kelime ekle — arama sonuçlarında öne çıkarsın"

## UI — Skor gösterimi

Üretilen içeriğin üstüne skor barı:
- **75-100:** Yeşil → "Harika listing! Pazaryerine hazır."
- **50-74:** Turuncu → "İyileştirilebilir" + eksik bilgi listesi + "Ücretsiz Yeniden Üret" butonu
- **0-49:** Kırmızı → "Eksik bilgi çok fazla" + eksik bilgi listesi + "Ücretsiz Yeniden Üret" butonu

Skor barı animasyonlu dolsun (0'dan hedefe, 1sn). Skor sayısı büyük fontla: **72**/100.

## Ücretsiz revize akışı (skor < 75)

- "Ücretsiz Yeniden Üret" butonu aktif
- Tıklayınca form'a geri dön, eksik alanlar highlight edilmiş
- Kullanıcı eksik bilgileri girer → ücretsiz üret
- Backend kontrol: `uretimId` ile ilişkili, `ucretsizRevize: true` flag
  - Kredi düşülmez
  - Sadece 1 kez ücretsiz hak

## API değişikliği

- Response'a `skor` ve `oneriler` alanları ekle
- `lib/listingSkor.ts` — Input: üretilen çıktı + kullanıcı inputları + platform kuralları → Output: `{ skor, oneriler, detay[] }`
- Ücretsiz revize endpoint'i: mevcut endpoint'e `ucretsizRevize: true` + `uretimId` parametresi
