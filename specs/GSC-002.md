# GSC-002 (P1) — /auth robots.txt'e ekle

> **Status:** Tamamlandı (2026-05-17). Commit: `2ca7d72` — `/auth` ve `/auth/` disallow eklendi.

## Bağlam

GSC Performance → Pages raporunda `https://www.yzliste.com/auth` — 0 click, **4 impression** görüldü.

`next.config.ts` satır 32'de `/auth → /` 308 redirect var — route yok, ama Google URL'i hâlâ crawl edip search'te gösteriyor. `robots.ts` disallow listesinde `/auth` eksik.

## Sorun

- Googlebot `/auth` URL'ini crawl ediyor → 308 → `/` geçiyor → ancak impression sayımı `/auth` URL'ine yansıyor
- Search result'ta `/auth` URL'i görünebiliyor — kullanıcı tıklasa redirect yiyor, kötü deneyim
- Crawl budget harcanıyor gereksiz yere

## Çözüm

`app/robots.ts` disallow listesine `/auth` ve `/auth/` ekle:

```ts
disallow: [
  "/api/",
  "/admin",
  "/hesap",
  "/odeme",
  "/giris",
  "/kayit",
  "/sifre-sifirla",
  "/profil",
  "/kredi-yukle",
  "/toplu",
  "/auth",      // ← ekle
  "/auth/",     // ← ekle (callback URL'ler için)
],
```

## Yapılacaklar

- `app/robots.ts` — disallow listesine `/auth` ve `/auth/` ekle
- deploy → `curl https://www.yzliste.com/robots.txt` ile doğrula

## Test

curl https://www.yzliste.com/robots.txt | grep auth

- robots.txt'te `Disallow: /auth` görünüyor
- 2-4 hafta içinde GSC Performance'da `/auth` impression kayboluyor
