### P0-2 — Cache-Control hatası

**Bulgu:** Hem anonim hem Googlebot user-agent ile yapılan tüm istek `Cache-Control: private, no-cache, no-store, max-age=0, must-revalidate` dönüyor.

```
curl -sI https://www.yzliste.com/ -A "Googlebot/2.1"
→ cache-control: private, no-cache, no-store, max-age=0, must-revalidate
```

`middleware.ts:128-131`'de anonim + public-cacheable durumda `public, max-age=0, must-revalidate` set edilmesi gerekiyor — ama bu satır hiç çalışmıyor görünüyor. Muhtemel sebepler:
- Supabase oturum cookie response'tan önce set edilince middleware response'u override ediliyor.
- Vercel'in default cache header'ı sonradan ezilmiyor.

**Etki:** Googlebot crawl bütçesini negatif etkiler. Bing daha katı — bu header'la sayfaları cache etmez ve nadiren tekrar gelir. CDN edge cache devre dışı kalıyor.

**Aksiyon:**
- BACKLOG'da `GSC-003 V2` olarak listelenmiş bu iş — "route-level `dynamic=force-static`" planı doğru. AUTH-UX-01 bekliyor. P2'den P0'a çıkar.
- Alternatif daha basit: middleware'de `response.headers.set` yerine `response.headers.append` veya `Headers` constructor ile yeniden yarat. Veya middleware mantığını next.config redirects yerine `headers()` config'e taşı (config'de `source: '/'`, `headers: [{key: 'Cache-Control', value: 'public, max-age=0, must-revalidate'}]`).
- Test: deploy sonrası `curl -sI https://www.yzliste.com/ -A "Googlebot/2.1"` ile `public` görmek gerekiyor.

**Tahmini etki:** Crawl frekansı 2-3x artar, indexed page count 1-2 ayda %15-25 büyür.

**Tahmini iş:** 2-4 saat.

### P0-4 — `/uret` Googlebot için içerik boş

**Bulgu:** `/uret` sayfası `"use client"` directive ile başlıyor. Googlebot'a giden HTML 79KB ama içeriğinin büyük kısmı (login banner, içerik üretim formu, platform chip'leri, içerik tipi kartları) HTML'de **yok** — sadece client-side render edildikten sonra görünüyor.

```bash

0
$ grep -c "3 kredi hediye" /tmp/yz-uret.html
```

- BACKLOG'da `GSC-007` olarak "kapatıldı — yanlış alarm" diye yazılmış. Ama bizim yaptığımız HTML kontrolü gösteriyor ki: indexli olabilir ama ranking için içerik boş.

### P1-3 — Title HTML entity bug

**Bulgu:** Blog post title'larında `'` apostrof `&#x27;` olarak render ediliyor:
```html
<title>Trendyol&#x27;da Satışları Artıran Listing Nasıl Yazılır?...</title>
```

OG description'larında da aynı:
```
content="Amazon&#x27;un dönüşüm odaklı A9 algoritmasını..."
```

Google tarayıcı title bar'ında doğru gösterir (entity decode edilir) ama bazı social platformlarda ve eski crawler'larda (Bing özellikle) `'` yerine `&#x27;` görünür.

**Aksiyon:** Markdown parser'da title alanını okurken HTML escape kapat. `app/blog/[slug]/page.tsx`'de `generateMetadata` içinde `title: yazi.baslik` doğrudan veriliyor — sorun parser'da string'in HTML escape edilmesinde. `String.prototype.normalize` veya `he` (html-entities) ile decode et.

BACKLOG'da `SEO-02` olarak listelendi.

**Tahmini iş:** 1-2 saat.
